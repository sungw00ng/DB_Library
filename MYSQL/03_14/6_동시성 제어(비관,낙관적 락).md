- Pessimistic Lock test (mysql 기본 값: REPEATABLE READ)
```sql
-- 세션 1
START TRANSACTION;
SELECT * FROM book WHERE book_seq = 50002 FOR UPDATE;
UPDATE book SET stock = stock - 1 WHERE book_seq = 50002;

-- 세션 2 (세션 1이 커밋 전)
START TRANSACTION;
SELECT * FROM book WHERE book_seq = 50002 FOR UPDATE;
```

- Optimistic Lock(낙관적 락) test (mysql 기본 값: REPEATABLE READ)
```sql
-- 1. 세션 1 START ~ UPDATE 실행 (커밋 X)
START TRANSACTION;
SELECT book_seq, stock, version FROM book WHERE book_seq = 50002;
UPDATE book SET stock = stock - 1, version = version + 1 
WHERE book_seq = 50002 AND version = 0;

-- 2. 세션 2 START ~ UPDATE 실행
START TRANSACTION;
SELECT book_seq, stock, version FROM book WHERE book_seq = 50002;
UPDATE book SET stock = stock - 1, version = version + 1 
WHERE book_seq = 50002 AND version = 0;

-- 3. 세션 1 COMMIT
COMMIT;

-- 4. 세션 2 ROW_COUNT 확인 → 0이면 충돌
SELECT ROW_COUNT();

ROW_COUNT()|
-----------+
          0|
-- 충돌 감지 (성공)

```

- 비교

| 구분    | 비관적 락                   | 낙관적 락                   |
| ----- | ----------------------- | ----------------------- |
| 방식    | 먼저 락을 걸고 작업             | 락 없이 작업 후 충돌 검사         |
| 구현 방법 | `SELECT ... FOR UPDATE` | `version` 컬럼 비교         |
| 충돌 처리 | DB가 다른 트랜잭션을 **대기시킴**   | 애플리케이션이 **충돌 감지 후 재시도** |
| 성능    | **충돌 많을 때 유리**          | **충돌 적을 때 유리**          |
| 데드락   | 발생 가능                   | 거의 없음                   |
